## 3. Ethical Risks

### 3.1 Fairness & Bias
- The project must not produce or reinforce discriminatory outputs against any group based on race, gender, religion, nationality, or other protected characteristics.
- Datasets and logic used should be reviewed for implicit bias.

### 3.2 Transparency
- The system's decision-making process should be understandable and explainable to users.
- Users should be informed about what the system does and does not do.

### 3.3 Accountability
- Clear ownership of the project is established (see LICENSE).
- Any errors, bugs, or harmful outputs should be reported and addressed promptly.

### 3.4 Privacy
- User data must be handled responsibly. See `docs/privacy-note.md` for details.

### 3.5 Misuse
- The project should not be used to deceive, manipulate, harass, or harm individuals.
- Developers commit to not deploying this system in high-risk or safety-critical contexts without further review.

---

## 4. Mitigation Strategies

- Conduct regular code and logic reviews with ethical considerations in mind.
- Document all assumptions made in design and implementation.
- Provide clear documentation so users understand system limitations.
- Respect all applicable laws and institutional policies.

---

## 5. Conclusion

This project was developed with awareness of its ethical responsibilities. The team commits to ongoing reflection and improvement on these issues throughout the project lifecycle.
